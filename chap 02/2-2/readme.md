# 실습과제 1

## 라이브러리는 헤더파일과 목적파일로 구성되어 있다.헤더파일과 목적파일에 들어 있는 내용이 무엇인지 각각 설명하시오.
* 헤더파일은 함수나 변수 등을 선언하거나 클래스나 타입을 정의한 것들을 포함하고 있으며, 컴파일러에게 정보를 넘겨주는 역할을 한다.
* 목적파일은 정의들을 컴파일 한 결과이며, 함수 본문의 기계어 번역본이다. 링커가 실제 코드를 찾아 연결하는 대상이다.

## 함수의 선언,정의,호출은 각각 어디에 존재하는가?
* 선언 : 헤더파일
* 정의 : 소스파일
* 호출 : 사용자 소스

## <iostream>헤더파일에 cin,cout은 extern변수로 선언되어 있다. extern 선언과 변수선언의 차이를 설명하라.그리고, 그럼 cin,cout의 변수선언은 어디에 있는가?(교재 2.5절 참고)
* 변수 선언은 객체를 생성하여 메모리 공간을 할당해주지만, extern 선언은 선언만 하고 메모리 공간을 할당하지 않는다.
* cin과 cout은 실제로는 Microsoft 런타임 라이브러리 dll에 있으며, 링크 단계에서 dll로 연결한다.

## 라이브러리 함수나 객체를 사용하기 전에 헤더파일을 include하는 이유를 설명하라.
* 컴파일 할 때에 라이브러리의 목적파일을 직접 들여다 볼 수 없으므로, 정의된 함수의 정보를 담고있는 헤더파일 없이 컴파일하게 되면, cout이나 cin이 무엇인지 알 수 없기 때문에 컴파일 오류가 발생한다.

# 실습과제 3
```
char number[8];
cout << "학번을 입력하세요 >> ";
cin.getline(number, sizeof(number), '\n');

char name[10];
cout << "이름을 입력하세요 >> ";
cin.getline(name, sizeof(name), '\n');

char address[100];
cout << "주소를 입력하세요 >> ";
cin.getline(address, sizeof(address), '\n');
```
* 각 내용을 할당받을 버퍼를 만들고 그 안에 키보드로부터 입력을 받아 버퍼를 채운다.
* ignore를 사용하면, 엔터를 두번 쳐야 하는 문제가 발생하였다. 그 이유로, getline에서 '\n'을 읽고 버퍼에서 제거까지 하여 발생했다.
```
cout << "1.학번:" << number << '\n';
cout << "2.이름:" << name << '\n';
cout << "3.주소:" << address << '\n';
```
* 입력받았던 내용이 담긴 버퍼를 하나씩 출력한다.

# 실습과제 4
```
double num, sum = 0;
```
* 기준이 되는 num = 0과 더한 값을 담아둘 sum을 선언
```
cout << "실수 5개를 입력하세요 >> ";
for (int i = 0; i < 5; i++) {
    cin >> num;
    if (num > 0)
        sum += num;
}
```
* 실수 5개를 키보드로부터 입력받는다. 'cin >>'은 공백을 자동으로 구분해준다.
* 0보다 큰 수(양수)일 경우, num에 더해준 뒤에 저장하는 것을 반복한다.
```
cout << "양수의 합은 " << sum << "입니다." << endl;
```
* 양수를 모두 더한 sum을 출력
